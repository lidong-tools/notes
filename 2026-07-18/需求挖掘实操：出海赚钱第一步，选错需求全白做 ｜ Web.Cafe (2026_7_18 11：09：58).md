# 需求挖掘实操：出海赚钱第一步，选错需求全白做

## 01 找需求这事儿，比你想的难得多

赫兹在复盘自己的出海经历时，说过一句话：

"选品定生死——你能不能成功，很大程度上已经在选需求那一刻决定了。"

不是技术不行，不是SEO不懂，不是域名没选好——是 一开始选错了需求，后面再怎么努力都是在填坑。

龙猫在21天航海里，前3天全花在找需求上，找到凌晨3点没找到合适的，差点崩溃。

赫兹自己更狠——上线20多个站，大部分没声响，直到第4个月才找到有起色的那个。

你可能会觉得，找个需求有什么难的？

打开谷歌，搜一下，不就知道用户在搜什么了吗？

问题是： 你看到的只是冰山一角。

谷歌搜索框下拉推荐的那几个词，几百万人都能看到。你去做了，竞争者已经排好队了。

真正的机会藏在数据里——藏在Semrush的筛选条件里，藏在Google Trends的曲线里，藏在竞品网站的出站链接里。

今天这篇文章，我就把"找需求"这件事从头到尾拆透。

## 02 先搞清楚：什么才是"好需求"

很多人搞混了一个概念——把"大需求"当成了"好需求"。

"Password Generator"这个关键词，月搜索量几十万，算大了吧？

但你去看它的KD（关键词难度），90+。排在首页的是那些做了10年的老站，DA（域名权重）80+。你一个新站，拿什么跟人家拼？

搜索量大 ≠ 值得做。值得做的标准是综合性价比。

哥飞总结了一个公式，我觉得是出海需求挖掘最重要的一个公式：

KDRoi = 搜索量 x CPC / KD

三个变量：

- 搜索量（Volume） ：有多少人在搜，代表需求大小
- CPC（Cost Per Click） ：广告主愿意为这个关键词付多少钱，代表商业价值
- KD（Keyword Difficulty） ：优化难度，代表竞争激烈程度

用这个公式算一下，你会发现一个反直觉的结论——

关键词A ：搜索量5000，CPC \$2，KD 25 → KDRoi = 400 关键词B ：搜索量50000，CPC \$0.1，KD 80 → KDRoi = 62.5

关键词B的搜索量是A的10倍，但KDRoi只有A的1/6。

关键词A才是你应该做的那个。

为什么？因为搜索量只有5000，竞争也不大，做上去的概率很高。而且CPC \$2说明广告主愿意花钱，意味着这个需求离钱很近，变现空间大。

离钱越近的关键词越值钱。这句话值得你记在笔记本上。

需求不是等来的，是挖出来的。我整理了8种方法，按难度从低到高排序，新手可以直接从第1种开始。

- Editor → Photo Editor、Video Editor、PDF Editor

- Template → Resume Template、Invoice Template、Contract Template

- Assistant → AI Assistant、Writing Assistant
- Simulator → Mortgage Simulator、Investment Simulator
- Scraper → Web Scraper、Data Scraper
- Comparator → Product Comparator、Price Comparator

具体怎么用？

1. 拿一个词根（比如 Generator）去 Semrush 查询
2. 设置筛选条件：KD 0-29，CPC 大于等于 \$0.1
3. 导出 CSV，用 KDRoi 公式计算每个词的得分
4. 按得分从高到低排序，排在最前面的就是最值得做的

一个词根就能挖出几十个甚至上百个候选需求，51个词根就是51片蓝海。

### 方法二：竞品出站链接发现需求

找到一个竞品网站后，用 Semrush 分析它的出站链接——它链接到哪些产品、哪些工具？

竞争对手认可的产品，就是他们验证过的需求。

操作路径：Semrush → 竞品域名 → Backlinks → Referring Domains → 分析对方链接了谁。

### 方法三：支付网关反查（直接看谁在赚钱）

通过 Stripe.com 等支付网关反查入站域名，找到那些 已经接入了支付、正在赚钱 的网站。

然后分析他们的关键词，找你还能切入的蓝海机会。

这个方法的厉害之处在于： 你看到的不是"可能赚钱"的需求，而是"已经有人赚到钱"的需求。

### 方法四：词根找词 → 站找词 → 站找站（赫兹核心循环）

这是赫兹反复强调的方法论：

- 词根找词 ：用 Generator/Converter/Maker 等词根扩展关键词
- 站找词 ：分析竞品网站的关键词来源（GSC/Semrush/Similarweb）
- 站找站 ：从一个站发现同类站，从同类站发现更多词

关键在于：这个循环用得越多，你的"网感"越强。赫兹说，做了几个月之后，他看到一个新词就能直觉判断值不值得做——这就是网感。

### 方法五：社媒热点追踪

- Twitter/X：关注 #SaaS #IndieHackers #BuildInPublic 标签
- Reddit：r/SaaS r/Entrepreneur r/SideProject 子版块
- TikTok：tiktok.com/discover/关键词

发现新需求 → 快速验证 → 上线网站。速度就是优势。

### 方法六：插件抱怨找需求

去 Chrome Web Store 找高下载量的插件 → 看差评 → 差评就是未被满足的需求。

高下载量 + 差评多 = 蓝海机会。

工具：chrome-extension-viewer.vercel.app

### 方法七：AI导航站找需求

theresanaiforthat.com/requests/ — 用户主动提交的AI工具诉求，就是真实需求。

### 方法八：Vercel子域名挖掘

查看别人部署在Vercel上的产品，发现新需求、新关键词。这个方法比较进阶，适合有一定经验后使用。

## 04 KDRoi公式实操：五分钟筛出黄金关键词

理论讲完了，手把手带你走一遍。

第一步：打开 Semrush Keyword Magic Tool

输入一个词根，比如 "Generator"。

第二步：设置筛选条件

- KD（Keyword Difficulty）：0-29 — 新手只看低难度的词
- Volume（搜索量）：200-10000 — 太小没意义，太大竞争太激烈
- CPC：大于等于 \$0.1 — 排除没有商业价值的词
- 排除 "near me" — 本地搜索词，工具站做不了

第三步：导出 CSV

把筛选后的关键词列表导出来。

第四步：计算 KDRoi

在 Excel 里新增一列，公式：

```javascript
KDRoi = Volume x CPC / KD
```

第五步：排序

按 KDRoi 从高到低排序，排在最前面的就是综合性价比最高的关键词。

举个例子，你可能会筛出这样的结果：

| 关键词 | 搜索量 | CPC | KD | KDRoi |
|---|---|---|---|---|
| AI Headshot Generator | 8100 | \$3.5 | 18 | 1575 |
| QR Code Generator | 45000 | \$0.8 | 72 | 500 |
| Password Generator | 110000 | \$1.2 | 92 | 1433 |
| Invoice Generator | 22000 | \$4.2 | 45 | 2053 |

你看，Password Generator 搜索量最大，但因为KD太高，KDRoi反而不如 Invoice Generator。

但注意——KD超过29的词新手直接跳过，所以Password Generator和Invoice Generator都不在考虑范围内。AI Headshot Generator 才是新手的首选：KDRoi 1575，KD只有18，做上去的概率很大。

这就是数据驱动决策的力量—— 用数据说话，不要用直觉。

## 05 需求验证四步法：走完再动手

找到高KDRoi的关键词后，别急着建站。先走完这四步验证：

![image.png](data:,)

### 第一步：Semrush 查搜索量和KD

确认这个关键词确实有搜索量，且竞争在你能力范围内。

判断标准：

- KD > 50 → 新手直接放弃
- 搜索量 < 100 → 除非CPC很高，否则不值得做
- 有搜索量 + 竞争低 = 进入下一步

### 第二步：Google Trends 看趋势

打开 trends.google.com，输入你的关键词。

看什么：

- 是不是季节性词汇？比如 "Christmas Card Generator" 只在12月有量，做了一年到头只有1个月能赚钱
- 趋势是在上升、平稳还是下降？上升趋势 > 平稳 > 下降趋势
- 对比不同国家的数据——有些词在美国有量，在印度没量，反之亦然

经验法则：选择全年稳定或上升趋势的词。

### 第三步：谷歌搜索看竞争

这是最关键的一步——直接去谷歌搜索你的关键词，看第一页都是什么网站。

看什么：

- 如果第一页全是 Wikipedia、Amazon、GitHub 这些巨头 → 放弃，你打不过
- 如果是一些不知名的小站、博客 → 机会来了
- 看看排名第一的站，DA多少？内容质量如何？你能做得比它更好吗？

问自己一个核心问题：如果我的网站出现在这个搜索结果里，用户会点进来吗？

如果答案是"会"，这个需求就值得做。

### 第四步：CPC看商业价值

回到 Semrush，看这个关键词的CPC。

判断标准：

- CPC > \$1 → 用户愿意花钱，广告收入高
- CPC $0.1-$ 1 → 有一定商业价值，可以赚广告费
- CPC < \$0.1 → 有流量但难变现，谨慎考虑

离钱越近的关键词越值钱。金融、法律、医疗、B2B软件类的关键词CPC通常很高，但竞争也大。工具类关键词（Generator、Converter、Calculator）是新手最好的切入点——CPC中等，KD可控。

## 06 新手最容易踩的4个坑

做需求挖掘这一步，新手最容易犯4个错误。我见过太多人在这些坑上浪费了数周甚至数月。

坑一：自嗨选需求

"我觉得这个需求应该很大。"

这句话是新手最常说的。问题是——你的直觉不重要，数据才重要。

你以为的巨大需求，可能搜索量只有几十。你以为没人关注的小众需求，可能月搜索量过万。

用数据说话，不要用直觉。每一个需求决策，都要有 Semrush 的数据支撑。

坑二：追大词

"我要做 Password Generator，搜索量11万！"

KD 92，新站毫无机会。

新手正确的策略是：保小图大。先做KD<29、搜索量几千的小词练手，站稳脚跟后，再逐步扩展到更大的词。就像打游戏，先在新手村打小怪攒经验升级。

坑三：忽略CPC

"这个词搜索量很大！"

但CPC只有\$0.05，意味着广告主不愿意为这个关键词花钱。你就算做到了首页，有流量也赚不到钱。

搜索量 x CPC 才是真实的商业价值。不要只看搜索量。

坑四：不做验证就动手

"先做站再说！"

做了一周，上线了，发现没搜索量。或者有搜索量但全是季节性词，一年只有1个月有流量。又或者CPC低到几乎没有变现空间。

验证四步法走完，只需要30分钟。但这30分钟能帮你省下1周的无效劳动。

## 07 好需求 vs 坏需求：一图看懂区别

![image.png](data:,)

好需求有什么特征？

- 搜索量稳定或增长，不是季节性
- KD < 30，新站有机会做上去
- CPC > \$0.5，有商业价值
- 搜索结果第一页没有巨头垄断
- 用户搜索意图明确（工具型 > 信息型）
- 可以用AI快速实现

坏需求有什么特征？

- 搜索量看起来大但KD极高
- CPC极低，流量无法变现
- 竞争对手全是DA 80+的老站
- 季节性太强，一年只有1-2个月有量
- 搜索意图模糊，用户到底想要什么搞不清楚
- 需要大量人工维护，无法自动化

## 08 找需求是一个循环，不是一次性的

最后说一个很多人忽略的认知——

找需求不是"第一步做完就再也不用管了"，而是一个 持续循环 的过程。

哥飞把这个叫做"种树"：你种下一棵树（做一个站），运营的过程中你会发现新的需求线索，于是再种一棵树，再发现新需求……

赫兹的真实路径就是这样的：前几个月上线20多个站大部分没声响，但每做一个站都在积累网感和数据，最终在第4个月找到有起色的那个。

那个有起色的站，不是运气好碰到的，是前面20多个"失败"的站堆出来的经验。

所以如果你刚开始做，前几个站没流量，别焦虑。这是正常的。

你需要做的是： 快速验证，快速失败，快速下一个。

找到一个需求 → 走验证四步法 → 值得做就建站 → 不值得就换下一个。每个需求的验证周期控制在2-3天，不要在一个明显不对的方向上浪费时间。

## 09 从今天开始，你可以做的3件事

如果你读到这里想动手了，这三件事今天就可以做：

1. 打开 Semrush，拿一个你感兴趣的词根（比如 Generator 或 Calculator）搜一搜，设置 KD 0-29 + CPC 大于等于 \$0.1 的筛选条件，感受一下数据。

没有 Semrush 账号的话，Google Ads 关键词规划师（免费）也能查搜索量和CPC。

2. 打开 Similarweb，查一个你常用的工具网站（比如 Canva、Notion），看看它的流量来源和关键词分布。

你会惊讶地发现，那些看起来"不怎么样"的网站，月访问量可能是几十万甚至上百万。

3. 打开 Google Trends，对比 "AI Logo Maker" 和 "AI Headshot Generator" 的趋势曲线。

看看哪个在上升，哪个在下降，感受一下数据驱动决策和直觉判断的差异。

从需求到上线，中间的技术环节其实没有你想的那么难。尤其是有了AI编程工具之后，很多以前需要几天才能做完的事，现在几小时就能搞定。

出海赚钱这件事，最大的门槛从来不是技术，是认知和执行力。

认知靠学习，执行力靠行动。
